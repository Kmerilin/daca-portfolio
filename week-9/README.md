# Nädal 9: Värbamisjuhend — Kairi GitHub portfoolio peer review

## 👤 Minu roll
Kaart A — Palkamisjuhi vaade (HR/Hiring Manager) + Kaart D — Tiimijuhi vaade (Team Lead), hindasin Kairi portfooliot

## 🎯 Äriprobleem
UrbanStyle kasvab kiiresti ja vajab palgata 7 uut andmeanalüütikut, kuid juhtkonnal (Liis, Toomas, Kristi, Anna, Marko) puudub selge kriteerium, kuidas GitHub-portfoolio põhjal kandidaate hinnata. Meeskond pidi oma 8-nädalase kogemuse põhjal koostama värbamisjuhendi.

## ⚙️ Lähenemine
Hindasin Kairi portfooliot kahest vaatenurgast. Palkamisjuhina vaatasin, kas portfoolio toetaks tugevat CV-d ja on "pin-worthy" LinkedIn jaoks. Tiimijuhina hindasin iseseisvust, kommunikatsiooni ja koostöövõimet. Mõlema rolli juures andsin 3 tugevust, 2 parandusettepanekut ja põhjendatud värbamissoovituse.

## 🔍 Peamised Leiud
- **Kaart A:** CV perspektiiv "Ei veel", LinkedIn perspektiiv "Ei ole pin-worthy" — peamine probleem on README vastuolud ja puuduvad mõju/tulemuse laused
- Tugevused: 105 commit'i ja järjepidevus, aus "minust" lugu (koondamine → ümberõpe), asjakohane tehnoloogiapinu junior DA rollile (PostgreSQL/Supabase, pandas, Power BI/Streamlit)
- **Kaart D:** iseseisvus "hea, kuid vajab paremat lõpetamist", kommunikatsioon "arendamist vajav" — README-d ei selgita tulemusi arusaadavalt, puuduvad lühikokkuvõtted
- Mõlemad rollid jõudsid sõltumatult sama värbamissoovituseni: **Lisavestlus** — potentsiaal on selgelt olemas, aga dokumentatsioon ei tõesta veel piisavalt, mis tegelikult commit'ide taga tehtud sai

## 💼 Äriline soovitus
Mõlemad minu rollid (palkamisjuht ja tiimijuht) jõudsid iseseisvalt sama järelduseni — README kvaliteet, mitte koodi olemasolu, on peamine takistus "Jah"-otsuseni. Soovitan UrbanStyle'i värbamisjuhendisse selge nõude: iga portfoolio projekt peab sisaldama lühikest "probleem → lahendus → tulemus" kokkuvõtet, mitte ainult koodi ja faile — vastasel juhul jääb tugev töö nähtamatuks isegi hea commit-ajalooga kandidaadil.

## 💡 Õpitu ja Väljakutsed
Õppisin vaatama sama portfooliot kahe erineva tööandja-läätse kaudu — palkamisjuhina keskendusin sellele, kas README ja projektid müüksid end iseseisvalt (CV/LinkedIn vaade), tiimijuhina aga sellele, kas dokumentatsioon annab aimu inimese iseseisvusest ja koostööst. Väljakutseks oli hoida hinnangud tasakaalustatuna ja konkreetsena — mitte üldsõnaline kiitus ega kriitika, vaid selged näited. Huvitav oli näha, et mõlemad rollid jõudsid sõltumatult sama järelduseni: potentsiaal on olemas, aga dokumentatsioon ei tõesta seda veel piisavalt.
